git基础-记录每次更新到仓库

在什么时候需要提交更新到仓库？

当我们获取到某个项目的git仓库后，在此基础上做开发，完成一定阶段的目标后，将更新提交到仓库。

工作目录的文件状态变化周期 文件状态变化周期相关的命令

  • 检查当前文件状态

    $ git status

  • 跟踪新文件

    $ git add README

  • 忽略某些文件

    $ cat .gitignore

    *.[oa] (忽略以.0或者.a结尾的文件)

    *. (忽略以~结尾的文件)

  • 查看已经暂存和未暂存的更新

    $ git status

    changes to be commited

    … 表示的是已经暂存的文件

    changes not staged for commit

    … 表示的是还未暂存的文件

  • 查看尚未暂存的文件更新了哪些部分

    $ git diff

  • 查看已经暂存的文件和上次提交的快照的差异

    $ git diff –cached

    注意上述两个命令都是针对当前最新的未暂存的或者最新的暂存的文件

  • 提交更新

    $ git commit

    $ git commit -m “content” 跟提交说明

  • 跳过使用暂存区域

    $ git commit -a

    git会自动把所有的已经跟踪过的文件一并提交，从而跳过git add,注意针对的是已经跟踪过的文件（跟踪过的文件，当被再次修改的时候，可以直接git commit -a）

  • 移除文件

    $ git rm + file

  • 移动文件

    $ git mv file_from file to
